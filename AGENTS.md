# Repository Instructions for Codex

- The site owner is not technical; keep explanations simple, clear, and free of unnecessary jargon.
- When the site owner asks for changes, make the requested edits directly in the repository rather than only explaining how to do them.
- After making edits, run an appropriate check when practical, then commit the changes back to the current branch.
- After changes are made, start the local dev server when practical and give the site owner the local preview URL, usually `http://127.0.0.1:4321/JFChiropractic`.
- In final responses, summarize what changed, mention any checks run, and call out anything that could not be completed.
